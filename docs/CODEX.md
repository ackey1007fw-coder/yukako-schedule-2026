# Codex への指示書 — データ更新担当

あなた（Codex）の担当は **「SNS投稿をサイトのデータに反映する」** こと。
新機能の実装・レイアウト変更・色の変更は担当外（Claude/Sonnet の領分）。頼まれていない範囲を触らない。

## 毎回の作業フロー

1. `AGENTS.md` を読む（特に**文体ガイド**と絶対ルール）。
2. `main` から作業ブランチを切る（例: `codex/news-2026-07-06`）。
3. 該当するデータファイルだけを編集する:
   - お知らせ → `src/data/news.ts`（配列の**先頭**に追加）
   - 本人のスナップ写真 → `src/data/photos.ts` + 画像配置
   - #ゆかJET関連の投稿・告知画像 → `src/data/gojetPromo.ts`
   - 公演・イベントの新規/変更 → `src/data/events.ts`
   - 過去出演のアーカイブ → `src/data/highlights.ts`
4. 画像があれば AGENTS.md「画像を追加する手順」どおりに
   `public/images/` 配置 → `node scripts/generate-responsive-images.mjs`。
5. `npx tsc --noEmit` と `npm run build` が通ることを確認。
6. PR を `main` 宛てに作成。タイトルは日本語で「何を追加したか」。

## 文章の書き方（最重要・過去に問題になった点）

サイトに載せる日本語は **AGENTS.md の文体ガイドに必ず従う**。
過去に「説明臭い」「レポート調」という指摘を受けている。具体的には:

- ❌ `優花子さんが〜したことを投稿。〜と伝えています。` ← この型を使わない
- ❌ `〜が告知されています` `〜が伝わります` `〜な投稿です` で文を締めない
- ❌ 本人の感情の解説（「楽しみが高まった」「愛着がにじむ」）を書かない
- ⭕ 投稿の中身を先頭に。短く。カギ括弧は原文どおりの引用のみ。

### news.ts のテンプレート
```ts
{
  date: "2026.7.6",
  label: "X",           // X / Instagram / TikTok / チケット / お知らせ
  text: "#ゆかJET 〇〇。△△も。",  // 中身ファースト・1〜2文・60字目安
  url: "https://x.com/..."
}
```

### 迷ったら
- 文章に自信がなければ、PR の説明に「文言レビュー希望」と書く（Claude が直す）。
- 事実関係が不確かな情報は**載せない**でオーナーに確認する。

## 写真の振り分け（間違えやすい）

| 画像の種類 | 置き場所 |
| :-- | :-- |
| 優花子さん本人が写ったスナップ・稽古写真 | `photos.ts`（フォトギャラリー） |
| フライヤー・相関図・タイムテーブル・チケット案内などの**告知画像** | `gojetPromo.ts`（#ゆかJET特集） |

**告知画像をフォトギャラリーに入れない**（過去に混入事故あり。CIのimage-guardも見ている）。

## やってはいけないこと
- コンポーネント（`src/components/`）の構造・スタイル変更（文言の修正は可）
- `tailwind.config.ts` / `src/index.css` の変更
- `imageManifest.ts` の手編集（自動生成）
- 既存画像の同名上書き
- 他エージェントの未マージPRと同じファイルへの変更（先にマージを待つ）
