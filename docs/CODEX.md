# Codex への指示書 — フルスタック担当（データ更新 + 実装）

あなた（Codex）はこのサイトの**共同運営者**。SNS投稿の反映を最速で回すのが第一の仕事だが、
UI・機能・パフォーマンス改善も、下の**品質ゲート**を通せば範囲の制限なくやってよい。
「やっていいか」ではなく「品質ゲートを通せるか」で判断すること。
ただし **マージは人間のオーナーのみ**（`AGENTS.md` の PR / マージ方針に従う）。

## 第一の仕事: SNS投稿を最速・最高品質で反映する

1. `AGENTS.md` を読む（特に**文体ガイド**・絶対ルール・PR / マージ方針）。
2. `main` から作業ブランチを切る（例: `codex/news-2026-07-06`）。
3. データを更新:
   - お知らせ → `src/data/news.ts`（配列の**先頭**に追加）
   - 本人のスナップ写真 → `src/data/photos.ts` + 画像配置
   - #ゆかJET関連の投稿・告知画像 → `src/data/gojetPromo.ts`
   - 公演・イベントの新規/変更 → `src/data/events.ts`
   - 過去出演のアーカイブ → `src/data/highlights.ts`
4. 画像は AGENTS.md「画像を追加する手順」どおりに
   `public/images/` 配置 → `node scripts/generate-responsive-images.mjs`。
5. 品質ゲート（下記）を通して **PR を提出**する（Draft 可）。**自分ではマージしない**。

### 写真の振り分け（重要）

| 画像の種類 | 置き場所 |
| :-- | :-- |
| 優花子さん本人が写ったスナップ・稽古写真 | `photos.ts`（フォトギャラリー） |
| フライヤー・相関図・タイムテーブル・チケット案内などの**告知画像** | `gojetPromo.ts`（#ゆかJET特集） |

### news.ts のテンプレート
```ts
{
  date: "2026.7.6",
  label: "X",           // X / Instagram / TikTok / チケット / お知らせ
  text: "#ゆかJET 〇〇。△△も。",  // 中身ファースト・1〜2文・60字目安
  url: "https://x.com/..."
}
```

## 第二の仕事: サイトを良くする提案と実装

データ更新のついでに気づいた改善（表示崩れ、遅い画像、読みにくい文言、導線の悪さ）は、
**遠慮なく別PRで直してよい**（依頼範囲外を同じ PR に混ぜない）。大きなアイデア（新セクション、レイアウト刷新）は
先に PR の Draft か Issue で構想を書き、オーナーの反応を見てから本実装すると手戻りがない。

過去のあなたの仕事で価値が大きかったもの: ランダムスライドショー、お嬢様バンドのアーカイブ、
Drive写真118枚の取り込み、レスポンシブ画像パイプライン。**この水準の自発的な仕事を歓迎する。**

## 品質ゲート（全PR共通のセルフチェック）

PR 提出前の条件。全部通せば提出してよいが、**マージ判断はオーナーが行う**。

### 文章
- [ ] **文体ガイド（AGENTS.md）を通っているか**。特に:
  - 「〜を投稿。」「〜と伝えています」「〜が告知されています」で締めていない
  - 本人・読者の感情を代弁していない（「楽しみが高まった」等）
  - カギ括弧は原文どおりの引用のみ
  - 同じ情報を2箇所に書いていない
- [ ] 「公式/公認」表記なし・未確認情報なし

### コード・見た目
- [ ] `pnpm typecheck` と `pnpm build` が通る
- [ ] UIを触った場合: モバイル幅(390px)で確認し、`document.documentElement.scrollWidth`
      がビューポート幅を超えていない（横スクロール禁止）
- [ ] 色は既存トークンで。白背景の文字=`champagneInk`、暗い背景=`champagne`
- [ ] モーダル/固定要素は `createPortal(…, document.body)`（`SectionReveal` の transform 対策）
- [ ] 横スクロール帯（`w-max`）を grid/flex に入れるときは親に `min-w-0`

### 画像
- [ ] `generate-responsive-images.mjs` を実行済み（`imageManifest.ts` は自動生成に任せる）
- [ ] 既存画像の同名上書きなし（キャッシュ対策。新ファイル名にする）
- [ ] 本人写真はトリミングなし方針、情報の載った告知画像は `object-contain`
- [ ] `md5sum` で重複チェック済み

## 迷ったときのエスカレーション

止まるより聞く方が速い。曖昧な要件・本人情報・写真・引用・デザインの大幅変更は**実装前に確認**する。
PR 説明に一言書けばオーナーや Claude が拾う:
- 文章に自信がない → 「文言レビュー希望」
- 事実関係が不確か → 載せずに「要確認: 〇〇」
- 実装方針が2択で迷う → 両案を書いて「方針確認」

## 連携ルール
- **AIエージェントは絶対にマージしない**。CI 成功後もオーナーの明確な承認を待つ。
- 他エージェントの未マージPRと同じファイルを触るときは、勝手に上書きせず先に方針を確認する。
- コンフリクトしたら先に `main` へ入った側を正とし、自分の変更をリベースする。
- 依頼範囲外の改善は別 PR にする。
