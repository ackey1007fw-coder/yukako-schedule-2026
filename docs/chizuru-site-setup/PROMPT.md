## 13．デザイン方針

### 全体イメージ

凛としている／知的／上品／現代的／透明感／モデルらしい洗練。
近寄りがたい高級感ではなく、人柄の温かさも感じるもの。若い女性向けサイトに寄りすぎない。
**芸能事務所の公式プロフィールと個人ポートフォリオの中間**。

### カラー

ホワイト／ブラック／チャコールグレー／淡いグレー／控えめなシルバーを基本に、
アクセントとして深いボルドー、ネイビー、または柔らかなピンク。
色数を増やしすぎず、**モノトーン中心**にしてください。
（姉妹サイトとの差別化のため、クリムゾン×ゴールドや里季サイトのピンク主体の配色は使わないでください。）

### タイポグラフィ

- 日本語本文は読みやすさを最優先
- 見出しはファッション誌のような洗練、英字見出しを補助的に使用
- 極端に細い文字はスマートフォンで読みにくいため避ける
- 文字サイズ、行間、コントラストを十分確保する

### 写真の見せ方

- 写真を過度に加工しない／顔や体型を歪めない
- 画質の低いSNS画像を無理に大きく引き伸ばさない
- 撮影者クレジットを表示できる構造にする
- alt 属性を設定する
- 本人の尊厳を損なう切り抜きを避ける
- **写真はトリミングしない方針を基本とする**（モバイルは全体表示、フライヤー等の告知画像はサムネイルでも `object-contain`）

**案A（姉妹サイトと統一・推奨）**: Vite + React + TypeScript + Tailwind CSS + Vercel
（オーナーの既存サイト `yukako-schedule-2026` 等と同じ構成。保守ノウハウを流用できる。`/api/*` を使う場合は ESM（`export default handler`）で書く）

**案B**: Next.js（App Router）+ TypeScript + Tailwind CSS + Vercel + 静的生成または ISR

どちらの場合も：

モバイルファースト／画像最適化（WebP 生成スクリプト等）／OGP 対応／構造化データ（JSON-LD `Person` 等）／サイトマップ／robots.txt／404ページ／エラーハンドリング／アクセシビリティ対応／Lighthouse を意識した実装

### データ管理

プロフィールや活動情報を JSX へ直接書き込まず、**JSON または TypeScript データとして分離**してください。
管理者がコード全体を触らなくても、**データ追加だけで活動歴やスケジュールを更新できる**ようにしてください。

```ts
type Activity = {
  id: string;
  title: string;
  date: string;          // ISO 8601
  endDate?: string;
  category:
    | "contest" | "model" | "magazine" | "web" | "youtube"
    | "tiktok" | "photo-session" | "event" | "social-action";
  summary: string;
  role?: string;
  organizer?: string;
  sourceUrl?: string;
  imageUrl?: string;
  imageCredit?: string;
  status: "upcoming" | "completed" | "unconfirmed";
  confidence: "verified" | "source-backed" | "needs-confirmation";
  lastVerifiedAt: string; // 情報確認日
};

type ProfileItem = {
  label: string;
  value: string;
  sourceUrl?: string;
  confidence: "verified" | "source-backed" | "needs-confirmation";
  publishable: boolean;
};

type Source = {
  id: string;
  title: string;
  publisher: string;
  url: string;
  publishedAt?: string;
  accessedAt: string;
  sourceType:
    | "official-profile" | "official-social" | "interview"
    | "media" | "video" | "event";
};
```

`publishable: false` または `confidence: "needs-confirmation"` のデータは**画面に表示しない**ロジックを必ず実装してください（データとしては保持し、要確認リストとして管理する）。

## 15．文体ガイド（サイトに載せる日本語すべてに適用）

- レポート調の「〜を投稿。」「〜と伝えています」を多用しない
- 本人の感想・気持ちを代弁しない（本人が語った言葉のみ、出典付きで引用）
- カギ括弧「」は**正確な引用のみ**に使う
- 「〜を確認できます」を連発しない
- 「公式」「公認」と書かない（非公式サイトであるため）
- 未確認情報を断定形で書かない
- 現在も継続中か確認できない活動は過去形で書く

ただし、**本人が正式に名乗っていない肩書を SEO 目的で追加しないでください**。

- 推奨タイトル例：`伊東千鶴 応援・活動アーカイブ｜プロフィール・出演情報`
- 推奨説明文例：`Sophian's Contest 2024準グランプリ・伊東千鶴さんのプロフィール、モデル・メディア活動、社会貢献活動、出演予定を紹介する非公式応援サイトです。`

## 17．情報更新方針

- 活動情報には必ず「情報確認日」（`lastVerifiedAt`）を持たせる
- SNS投稿や外部ページが削除された場合、本人に不利益となる形でコピーを残さない
- 確認できなくなった情報は：非公開にする／`needs-confirmation` へ変更する／出典切れとしてデータ内で保持する
- 「2026年現在も活動中」などの表現は、最新情報が確認できる場合だけ使用する

## 21．最重要ルール

- 本人への敬意を最優先する
- 非公式サイトであることを明記する
- 未確認情報を断定しない
- 私生活を詮索しない／恋愛関係を推測しない
- DMや個人的な情報を掲載しない
- いじめや美容医療を見世物にしない
- 子どもや支援対象者の個人情報を守る
- 画像を無断転載しない／顔をAI生成しない
- 活動実績を誇張しない
- 「モデル」「タレント」「JJモデル」などの肩書は、本人の正式な表記に合わせる
- 現在も継続中か確認できない活動は過去形で書く
- 本人が削除した情報をアーカイブとして復元しない
- 外見だけでなく、言葉、学び、社会貢献、継続力、自己表現を同じ重さで伝える
- 姉妹サイト（優花子・里季・まこ）の人物・データ・画像を混入させない

最終的に、

1. 初めて伊東千鶴さんを知った人が、人物像と活動を正確に理解できる
2. ファンが最新情報を確認できる
3. 仕事関係者が実績や魅力を把握できる
4. 本人にとっても不利益や負担にならない

という四つを満たすサイトにしてください。
