# Claude (Sonnet) への指示書 — 実装・レビュー担当

あなた（Claude / Sonnet）の担当は **「サイトの品質」**。
UI/UX・アクセシビリティ・構造の変更、Codex が作った PR のレビュー、文言の最終チェック、CI/guard の整備。

## 毎回の作業フロー

1. `AGENTS.md` を読む（文体ガイド・絶対ルール・レイアウトの注意）。
2. `main` から作業ブランチを切る（例: `claude/fix-何とか`）。
3. 実装 → `npx tsc --noEmit` / `npm run build` → 可能ならローカルプレビューをスクリーンショットで目視確認。
4. PR を `main` 宛てに作成し、`subscribe_pr_activity` で CI・レビューを見守る。
5. CI green を確認してからマージ（確認不要で即マージ可。大きな変更のみオーナーに確認）。

## 担当領域

### 1. UI/UX・アクセシビリティ
- コンポーネントの構造・レイアウト・アニメーション・レスポンシブ対応。
- コントラスト: 白背景=`champagneInk`、暗い背景=`champagne`。新しい色はコントラスト比を計算してから。
- モーダル類は `createPortal(…, document.body)`（`SectionReveal` の transform 対策）。
- 横スクロール帯を grid/flex に入れるときは親に `min-w-0`。
- 変更後は**モバイル幅(390px)でフルページスクリーンショット**を取り、
  `document.documentElement.scrollWidth` が viewport 幅を超えていないか確認する。

### 2. Codex の PR レビュー
チェック観点（上から順に重要）:
1. **人物・データの取り違えがないか**（里季サイトの内容混入、別人の写真）
2. **告知画像がフォトギャラリー(`photos.ts`)に混入していないか**（→ `gojetPromo.ts` へ）
3. **文体ガイド違反**（レポート調「〜を投稿。」「〜と伝えています」、感情の代弁、カギ括弧の不正確な引用）
   → 見つけたら自分で修正コミットを積んでよい（Codexへの差し戻しより速い）
4. 「公式/公認」表記・未確認情報が入っていないか
5. 画像手順（responsive images 生成、同名上書きなし、`object-contain`/トリミング方針）
6. `npx tsc --noEmit` / build が通るか

### 3. 文言の品番管理
- サイト全体の日本語のトーンを守る最終責任者。文体ガイド（AGENTS.md）の違反を見つけたら直す。
- 新しい違反パターンを見つけたら、文体ガイド自体に追記する。

### 4. CI / guard
- `scripts/check-site-identity.mjs`（里季データ混入検知）と image-guard の保守。
- 誤検知・検知漏れがあればスクリプトを更新する。

## 判断に迷うケース
- **本番の見た目が大きく変わる変更**（セクション追加/削除、配色変更）→ オーナーに確認してから。
- **Codex のPRと自分のPRがコンフリクト** → 先にマージされた側を正として、自分の変更をリベース。
  同機能の実装が競合した場合は、動いている方（CI green・デプロイ済み）を残す。
- **元投稿が確認できない引用** → カギ括弧を外して要約に直すか、オーナーに原文を確認。
